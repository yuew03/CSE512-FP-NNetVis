Abstract



Deep learning is a class of machine learning algorithm which can take raw features of vastly large annotated datasets and use them to build predictive tools via the patterns buried inside. Due to the extreme size of the data, it is challenging for users to understand the “internal workings” of the trained network. Collaboration with Dr. Callin Switzer at eScience Institute and Department of Biology at the University of Washington, we will be attempting to visualize his trained feedforward neural network for prediction of non-linear flight dynamics of Manduca sexta. We will be using a dataset that contains sets of weights that shows the strength of each neural path, and our goal is to design a software that allows researchers to generate static/interactive visualization tool that effectively gains readers/users attention to the stronger neural paths selectively within millions of network edges.


We will develop new features based on the basic structure of Neural Network graph. We envision to first implement “Parallel Coordinate” as a form of our visualization and iteratively develop various features and encodings (e.g., interactive slide bar for thresholding weight of our interest, hover tooltips for detailed information about each node and edge) as we proceed. In order to better understand the characteristics of datasets, we will also perform exploratory data analysis by visualizing local(layer) and global(network) distribution of weight profile. We believe this is necessary for effective visualization of our dataset since it contains millions of weight profile. Based on our exploratory data analysis, we will carefully strategize data cleanup and visual implementations to provide better insights on “internal workings” of neural networks.

Running Instructions

links
- Using the slide bar with longer range to set the range of weights you do not interest, the links with that weights will be removed in the neural network;
- If you are interested in a smaller range of weights, which is difficult to set using the first slider bar,the seconde bar can help you set a smaller range of weights to exclude;
Bias node
- Click the button "Bias Nodes" to display the bias nodes for each layer;
- Click the button "No Bias" to delete the bias nodes for each layer;
Input/Output nodes
- Click the input/output node and the links connected the node and satisfy the range in the slide bar will be highlighted;
- Description for each node in this neural network is shown below;
Progress bar
- Dense refers to the links in between two layers. For example, Dense1 represents links between the input nodes and the first hidden layer;
- The number displayed after each dense progress bar refers to the amount of links that strisfy the slider range in each dense;
- The percentage displayed under each dense progress bar refers to the percentage of links that strisfy the slider range in each dense; For example, if these two slide bars are in the default value, there will be 430 links (links from bias node is not included) in dense1. If we delete the links with the weight in [-1, 0],the number of links in dense1 will be 233 and the percentage will be 233/430 * 100%, which is 54.19%.
- The progress bar named as Selected will be activited when you click on either the input node or the output node. After clicking, links connected to this node will be highlighted in green and in the same time, these selected process bar will display the details for the node that we clicked.
- The number displayed after each selected progress bar refers to the amount of links that strisfy the slider range and connected to the clicked node in each dense;
- The percentage displayed under each dense progress bar refers to the percentage of links that connected to the clicked node as well as strisfy the slider range in each dense; For example, if these two slide bars are in the default value, there will be 430 links (links from bias node is not included) in dense1. If we delete the links with the weight in [-1, 0] and click on the first input node, the number of links selected in dense1 will be 24 and the percentage will be 24/430 * 100%, which is 5.58%.

Input Nodes

\(\dot{\theta_0}\): starting head angular velocity (rad/s)

\(\dot{\phi_0}\): starting abdomen angular velocity (rad/s)

\(\dot{y_0}\): starting vertical velocity (cm/s)

\(\dot{x_0}\): starting horizontal velocity (cm/s)

\(\theta_f\): final head angle (rad)

\(\phi_f\): final abdomen angle (rad)

\(y_f\): final vertical position (cm)

\(x_f\): final horizontal position (cm)

\(\theta_0\): starting head angle (rad)

\(\phi_0\): starting abdomen angle (rad)

Output Nodes

\(\dot{\theta_f}\): final head angular velocity (rad/s)

\(\dot{\phi_f}\): final abdomen angular velocity

\(\dot{y_f}\): final vertical velocity (cm/s)

\(\dot{x_f}\): final horizontal velocity (cm/s)

\(\tau\): torque (g*cm^2/s^2)

\(\mathit{F_y}\): vertical force (g*cm/s^2)

\(\mathit{F_x}\): horizontal force (g*cm/s^2)
